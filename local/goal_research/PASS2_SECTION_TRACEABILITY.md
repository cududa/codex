## Target Home Reading Notes

A target listed in `Proposed home` means the row must be considered when
designing that interface. It is not an ownership classification by itself.

For Pass 2B:

- `T-EXT` owns extension lifecycle, configuration, and reachability; it is
  shared where extension mutation can create cadence work and pointer-only to
  final-input authority.
- `T-SHIM` owns demolition and removal terrain; it does not own cadence or
  final-input semantics.
- `T-TEST-PREP` owns prep sequencing, upstream baseline restoration,
  replacement matrix, and snapshots; it collects proof obligations but must not
  own behavior.
- `T-READINESS` owns readiness and handoff criteria only; checklist rows are
  not behavior ownership.
- `NAV-README`, `GLOSSARY`, and `OP-AGENTS` are navigation, vocabulary, and
  operational surfaces only.

## Status Key

| Status | Meaning |
| --- | --- |
| `Open` | Section has been inventoried but not rewritten. |
| `Split` | Section is expected to feed more than one target interface. |
| `Canonicalize` | Repeated authority should become canonical text plus explicit pointers. |
| `Leave` | Section should remain in its existing operational/navigation role. |
| `Review debt` | Section needs special fidelity review before rewrite closure; it is not by itself a claim that behavior is unresolved. |

Before treating a row as unresolved, classify the debt:

- Authority-settled rows should be rewritten deterministically from the source
  contracts, with no lingering ambiguity prose.
- Implementation details clarified outside `local/goal_research` must first be
  written into the applicable source authority doc before this traceability
  inventory treats them as settled.
- Target-home ownership questions may remain as owner/shared/pointer-only
  routing work without reopening behavior.
- Plan/authority conflicts must stop and fix either the source authority doc
  or the plan explicitly.
- True design gaps are design debt, not successor-doc prose.
